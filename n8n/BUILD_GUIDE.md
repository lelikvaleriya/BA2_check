# Ручная сборка воркфлоу в n8n (на случай если импорт workflow.json не сработает)

`workflow.json` можно попробовать импортировать напрямую (Workflows → знак "..." → Import from File).
Если что-то не подошло по версии n8n — соберите те же узлы вручную по этому гайду.
Заодно это даст понимание каждого узла для скриншотов и защиты проекта.

⚠️ **Узел чтения файлов с диска (шаг 3 ниже) не входит в `workflow.json`** — тип этого
узла зависит от версии n8n, поэтому его нужно добавить вручную в уже собранный воркфлоу
(замените им старый узел "Вставить .cha файл", если импортировали `workflow.json` раньше).
Остальные узлы (4-6) можно обновить, просто вставив в них актуальный код из файлов
`n8n/*.js` — они уже умеют обрабатывать сразу много файлов.

## Узел 1: Manual Trigger
Добавить узел → раздел "Trigger" → **Manual Trigger** ("When clicking 'Execute workflow'").
Параметров нет.

## Узел 2: Set — "Ключи Yandex Cloud (НЕ публиковать в скриншотах!)"
Добавить узел → **Edit Fields (Set)**. Переименовать узел (двойной клик на имя) — имя
должно совпадать буквально с тем, что использует код в узле 5.
Добавить два поля (оба String):
- `apiKey` — ваш API-ключ Yandex Cloud
- `folderId` — ID каталога (folder) в Yandex Cloud, где создан ключ
⚠️ Когда будете делать скриншоты для сдачи проекта — скройте значения этих полей
(в n8n есть кнопка "глаз" рядом с полем, чтобы спрятать значение) или заблюрьте на скриншоте.

## Узел 3: чтение файлов из папки (вместо ручной вставки)
Раз n8n у вас локальный, он может читать файлы прямо с диска — можно обработать
сразу всю папку с .cha файлами, без копипаста по одному.

Добавить узел → найти в поиске **"Read/Write Files from Disk"** (в старых версиях n8n
называется **"Read Binary Files"** — берите тот, что найдётся у вас).
Параметры:
- Operation: **Read**
- File(s) Selector — путь с маской, например:
  `C:\Users\user\Documents\Проекты\продвинутый_ИИ_проект\materials\examples\Дмитрий Фролов\Часть II. Проверка\Проверенные файлы\**\*.cha`

Узел прочитает все файлы, подходящие под маску, и отдаст по одному item на файл —
дальше вся цепочка (узлы 4-6) уже сама обрабатывает произвольное число файлов сразу.

Чтобы обработать другую папку (например, реальные файлы Анны/Марии, а потом — сырой
BA2) — просто поменяйте путь в этом одном поле.

> Для точечного теста одного файла — тот же узел, просто маска указывает на один
> конкретный файл вместо `**\*.cha`.

## Узел 4: Code — "Правила: найти подозрительные предложения"
Добавить узел → **Code**. Mode: **Run Once for All Items**. Language: **JavaScript**.
Скопировать в поле кода содержимое файла `n8n/rules_check.js` целиком.

## Узел 5: Code — "LLM: проверить и исправить подозрения"
Ещё один **Code**, тоже Run Once for All Items, JavaScript.
Скопировать содержимое `n8n/llm_recheck.js`.
Важно: этот узел обращается к узлу 2 по имени
(`$('Ключи Yandex Cloud (НЕ публиковать в скриншотах!)')`) —
имя узла в n8n должно совпадать буквально, иначе будет ошибка "node not found".

## Узел 6: Code — "Собрать результат"
Ещё один **Code**, Run Once for All Items, JavaScript.
Скопировать содержимое `n8n/rebuild_and_report.js`.
Этот узел тоже ссылается на узел по имени — `$('Правила: найти подозрительные предложения')`.

## Соединения (стрелки между узлами)
Просто соедините узлы линейно в порядке 1 → 2 → 3 → 4 → 5 → 6 (перетаскиванием от точки
на правом краю одного узла к левому краю следующего).

## Как посмотреть результат
После запуска (кнопка "Execute Workflow" на узле 1) кликните на узел
"Собрать результат" → вкладка **"Binary"**. Там будет по два файла
(`correctedFile`, `commentsFile`) **на каждый обработанный файл** — у каждого кнопка
"Download". При обработке целой папки узел выполнится один раз, но вернёт много items —
переключайтесь между ними стрелочками вверху панели вывода.

⚠️ На папке из многих файлов вызовы к YandexGPT идут по очереди (один за другим), это
может занять несколько минут — не бага, а нормальная работа последовательных запросов.

## Апгрейд: настоящий RAG (векторный поиск), опционально
Сейчас узел 5 сам подбирает нужный кусок инструкции по названию сработавшего правила —
это простой и надёжный вариант "retrieval + augmented generation", но без embeddings.
Если хотите получить бонусный балл за RAG в более "классическом" виде:
1. Добавьте узлы для эмбеддингов (у Yandex Cloud есть модель `text-search-doc/latest`
   через тот же Foundation Models API — можно вызвать через HTTP Request по аналогии
   с `llm_recheck.js`, либо взять готовый node "Embeddings OpenAI", если он у вас есть)
   + **Default Data Loader** + **Simple Vector Store** (режим Insert), загрузите туда
   `n8n/knowledge_base.md` (один прогон, отдельно).
2. Замените узел 5 на **AI Agent** с этим Vector Store, подключённым как **Tool**
   (режим Retrieve As Tool) — модель сама будет решать, когда искать в базе.
Это красивее выглядит на скриншотах ("настоящий" RAG-узел), но и менее надёжно для
первого запуска — рекомендую сначала сдать рабочую версию с узлом 5 как есть.

## Апгрейд: автоматически брать файлы из вашей папки на Google Drive
Вместо ручной вставки текста в узел 3 можно подключить узел **Google Drive** (Download file)
и указать ID файла из вашей папки — тогда воркфлоу будет обрабатывать файлы прямо из
папки мастерской без копипаста. Это тоже хорошо смотрится как "плюс за креатив"
(внешний инструмент), но требует настройки OAuth-доступа к Google Drive в n8n.
