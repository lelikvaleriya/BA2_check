# Политика обработки данных проекта


## Ориентир: 152-ФЗ «О персональных данных»

Расшифровки детской речи с реальными именами (до анонимизации) — персональные данные
несовершеннолетних. Два положения 152-ФЗ, на которые опираемся:

- **Ст. 18, ч. 5 (локализация)**: запись, систематизация, накопление, хранение,
  уточнение, извлечение персональных данных граждан РФ должны вестись с использованием
  баз данных, физически находящихся на территории РФ.
- **Ст. 12 (трансграничная передача)**: передача персональных данных в страны без
  «адекватной защиты» (США, где размещена инфраструктура Anthropic/OpenAI, в этот
  список не входит) требует отдельного согласия субъекта (для ребёнка — законного
  представителя) именно на трансграничную передачу.

## Что это значит на практике

**Уже опубликованные данные CHILDES с разрешением родителей на реальные имена**
(то, что мы использовали для тестов — `T14-checked.cha` и другие файлы в
`materials/examples`) — вне периметра этой политики. Это открытые данные, их можно
свободно использовать в разработке и тестировании, в том числе через Claude.

**Новые, ещё не анонимизированные расшифровки детей** (то, что появится в будущем
до прохождения анонимизации) — под строгим правилом:

1. **Продакшн-обработка** — только через локальный n8n (self-hosted, не n8n Cloud) +
   YandexGPT (Yandex Cloud, российская юрисдикция). Уже так и есть по конструкции —
   Claude/OpenAI в самом воркфлоу не участвуют.
2. **Разработка и отладка с участием Claude** — реальные новые расшифровки в чат с
   Claude Code **не вставлять и не давать читать**. Для отладки новых правил/промптов
   использовать либо уже открытые файлы CHILDES (как сейчас), либо синтетические
   тестовые .cha-файлы, придуманные вручную, без реальных детей.
3. Если для нового пайплайна (проверка сырого BA2, см. `plan.md` → Roadmap) понадобится
   что-то куда-то сохранять (не просто обрабатывать на лету) — база должна физически
   находиться в РФ (ст. 18, ч. 5) — например, в самой Yandex Cloud, а не в стороннем
   зарубежном хранилище.

## Что стоит дополнительно проверить (не входит в текущий пайплайн, но по смыслу рядом)
- Политику Yandex Cloud по хранению/использованию данных, отправленных в Foundation
  Models API (логирование, дообучение на пользовательских данных и т.п.) — обычно
  описано в договоре/условиях использования Yandex Cloud.
- Формулировку согласия родителей — покрывает ли она автоматизированную обработку
  (в т.ч. ИИ), а не только сбор и хранение.
