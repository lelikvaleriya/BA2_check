# С нуля: n8n + YandexGPT + первый запуск агента

Рассчитано на то, что вы никогда не работали ни с n8n, ни с API языковых моделей.
Идём по аналогии с вашим же письмом волонтёрам: сначала пробуем ОДИН файл, потом остальные.

## Шаг 1. n8n — установлен локально, бесплатно навсегда

Node.js и n8n уже установлены на этом компьютере (сделано в этой сессии).
n8n НЕ требует постоянного облачного хостинга — воркфлоу запускается вручную,
поэтому "поднимать навсегда" не нужно: открыли, проверили файл, закрыли.

**Как запустить n8n** (каждый раз, когда хотите с ним поработать):
1. Откройте PowerShell / терминал.
2. Выполните: `npx n8n start`
3. Дождитесь строки `n8n ready on ::, port 5678`.
4. Откройте в браузере: **http://localhost:5678**
5. При первом заходе n8n попросит создать локальный аккаунт владельца
   (просто email + пароль, всё хранится только на вашем компьютере, никуда не уходит).

**Как остановить**: закрыть терминал или нажать Ctrl+C в нём.
Все воркфлоу и данные сохраняются между запусками (локальная база в `~/.n8n`).

<details>
<summary>Если вместо этого хотите n8n Cloud (пробный период)</summary>

1. Откройте https://n8n.io в браузере → "Get started" / "Try n8n cloud" → регистрация
   по email (доступен бесплатный пробный период, потом платно).
2. После регистрации откроется рабочее пространство ("workspace") — это и есть ваш n8n.
3. Слева — раздел **Workflows**. Нажмите "Add workflow" (создать пустой воркфлоу).

## Шаг 2. Ключ Yandex Cloud (YandexGPT)
1. Зайдите на https://console.cloud.yandex.ru → войдите через Яндекс ID (обычная
   российская почта/номер, без иностранной карты).
2. При первом входе Yandex Cloud создаст платёжный аккаунт и обычно даёт **грант**
   (пробные деньги на баланс) — этого более чем достаточно для проверки десятков файлов.
   Если гранта нет — привязать российскую карту тоже можно, стоимость запроса очень мала.
3. Создайте (или откройте существующий) **каталог (folder)** — он уже есть по умолчанию
   после регистрации. Скопируйте его **Folder ID** — это строка вида `b1g...`, видна
   в консоли рядом с названием каталога или в адресной строке.
4. Слева меню каталога → **Сервисные аккаунты** → "Создать сервисный аккаунт" → дайте
   любое имя → роль **`ai.languageModels.user`** (или `editor`) → создать.
5. Откройте созданный сервисный аккаунт → вкладка **"API-ключи"** → "Создать новый ключ" →
   скопируйте значение — **показывается только один раз**, сохраните в надёжном месте
   (например, менеджер паролей), не публикуйте его нигде.
6. Убедитесь, что в каталоге включён сервис **"Yandex Foundation Models"** (обычно включён
   по умолчанию, иначе включается в каталоге сервисов одной кнопкой).

В итоге у вас должно быть два значения: **API-ключ** и **Folder ID** — оба понадобятся
в узле "Ключи Yandex Cloud" в воркфлоу.

## Шаг 3. Собрать воркфлоу
Два варианта — выберите любой:
- **Импорт**: в n8n откройте созданный пустой воркфлоу → "..." (три точки, справа сверху)
  → "Import from File" → выберите `n8n/workflow.json` из этого проекта.
- **Вручную**: следуйте `n8n/BUILD_GUIDE.md` — там пошагово, какие узлы добавить и что в них вписать.

После импорта/сборки:
1. Откройте узел **"Ключи Yandex Cloud (НЕ публиковать в скриншотах!)"** → вставьте
   свой API-ключ в поле `apiKey` и Folder ID в поле `folderId` вместо плейсхолдеров.
2. Откройте узел **"Вставить .cha файл"** → в поле `chaText` вставьте содержимое
   тестового файла, например `materials/examples/Дмитрий Фролов/Часть II. Проверка/
   Проверенные файлы/1 набор/T14-checked.cha` (откройте его в Блокноте, скопируйте всё).
   В поле `fileName` впишите `T14-checked.cha`.

## Шаг 4. Первый запуск — один файл
1. Нажмите **"Execute Workflow"** (обычно кнопка внизу холста или на узле "Старт").
2. Дождитесь, пока все узлы выполнятся (зелёные галочки на каждом узле).
3. Откройте узел **"Правила: найти подозрительные предложения"** → посмотрите поле
   `flaggedCount` — сколько предложений вызвали подозрение (для T14-checked.cha
   должно быть около 5).
4. Откройте узел **"Собрать результат"** → там будет:
   - `correctedCha` — полный текст исправленного файла
   - `commentsMd` — что именно исправлено и почему
5. Скопируйте оба поля и сохраните как обычные текстовые файлы на компьютере
   (например, через "Скопировать значение" по клику на поле в выводе узла).

Если на этом шаге что-то не работает — самая частая причина: неверно скопированное
имя узла в коде (`$('Ключи Yandex Cloud...')` должно совпадать с реальным названием узла
буквально, включая скобки и пробелы) — проверьте название узла 2.
Ещё частая ошибка Yandex Cloud API — 401/403: не хватает роли у сервисного аккаунта
(нужна `ai.languageModels.user`) или не включён сервис Foundation Models в каталоге.

## Шаг 5. Дальше
Когда первый файл прошёл нормально — можно брать любой другой .cha файл из папки
`materials/examples` (или из вашей рабочей папки на Google Drive) и повторять шаг 4.

## Что сдавать по критериям курса (Трек 2)
- скриншоты воркфлоу со всеми узлами (спрятав/заблюрив значения apiKey и folderId!)
- краткое описание: перепроверка морфологической/синтаксической разметки корпуса
  детской речи CHILDES; узлы — Set (вход), Code (правила), Code (LLM + мини-RAG по
  базе знаний из инструкций курса), Code (сборка результата); модель — YandexGPT
  (Yandex Cloud Foundation Models) — платформа прямо разрешена заданием курса
- при желании — бонусный балл за RAG/креатив: см. раздел "Апгрейд" в `n8n/BUILD_GUIDE.md`
