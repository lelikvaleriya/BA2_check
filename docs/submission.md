# Материалы для сдачи (Трек 2: ИИ-агент)

## Краткое описание (3-5 предложений — то, что просит задание курса)

Агент перепроверяет и обогащает морфологическую и синтаксическую разметку расшифровок
детской речи в корпусе CHILDES (формат .cha, конвенции Universal Dependencies/Taiga), с
которым я работаю как координатор волонтёров. Воркфлоу в n8n читает папку файлов с диска
и для каждого: детерминированные правила (Code, JavaScript, 17 категорий — от пропущенных
род/число/падеж до опечаток в тегах разметки) находят подозрительные предложения →
языковая модель YandexGPT (Yandex Cloud Foundation Models) с мини-базой знаний из
инструкций курса подтверждает или отклоняет каждое подозрение, предлагает исправление и
исправляет опечатки расшифровщика прямо в тексте реплики → отдельный детерминированный
проход без LLM подтягивает согласованность повторяющихся слов по всему файлу → финальный
узел собирает исправленный .cha файл и отчёт с комментариями, включая отдельный список
мест, которые не удалось надёжно подтвердить автоматически (для точечной ручной проверки
перед публикацией). На тестовом файле (55 предложений на пробном файле, 960+ на пачке
из 55 файлов) агент находит и корректно исправляет подавляющее большинство подозрений,
включая опечатки набора текста, несогласованность разметки и структурные ошибки (ROOT,
согласование падежей).

## Что сдаётся
- Скриншоты воркфлоу в n8n со всеми узлами (значения `apiKey`/`folderId` скрыты)
- Это описание (выше)
- Модель: YandexGPT — платформа прямо разрешена заданием курса (Яндекс Клауд)

## Что сверх минимума (кандидат на «плюс балл за креатив»)
- **Мини-RAG**: узел LLM подтягивает релевантный кусок инструкций курса по названию
  сработавшего правила, а не гоняет модель по всему тексту инструкций вслепую
- **Самопроверка результата**: после ответа модели код сам (без LLM) перепроверяет,
  действительно ли решена заявленная проблема; если нет — один точечный повторный запрос
- **Проверка согласованности без LLM**: повторяющиеся слова в файле (например имя)
  получают одинаковую разметку по всему файлу, даже если модель обработала предложения
  по отдельности и не обязана была это помнить
- **Честная отчётность**: отдельный раздел «Требует ручной проверки» — там, где
  автоматика не смогла подтвердить исправление, вместо того чтобы тихо промолчать

## Полный контекст (для себя / если спросят на защите)

Корпус детской речи проходит несколько этапов ручной проверки: сначала Batchalign (BA2)
автоматически размечает сырую расшифровку, затем один студент вручную размечает/исправляет
эту разметку с нуля (Часть I), затем другой студент перепроверяет то, что сделал первый
(Часть II). Даже после двух ручных проходов остаются систематические ошибки — это видно
по документам «Типичные ошибки», которые сами студенты вели по ходу проверки.

Текущая версия агента работает на границе после Часть II — перепроверяет то, что уже
перепроверил человек, и подтверждаемо находит реальные, ранее пропущенные ошибки
(валидировано на 44 файлах одного из проверяющих и на всём корпусе — 576 файлов, 156k
предложений разбора без единого сбоя парсера).

Следующий практический шаг (за рамками сдачи курса, но уже спроектирован) — перевести
агента на вход сразу после BA2, то есть перепроверять сырую автоматическую разметку
напрямую, замещая часть ручной работы студентов на Части I. Профиль ошибок там гораздо
плотнее и другой по характеру (почти всегда самозамкнутый ROOT, междометия массово
принимаются за существительные/глаголы), поэтому архитектура для этого режима будет
гибридной: часть ошибок чинится механически без LLM, остальное — моделью, но уже на
каждое предложение, а не на редкие подозрения.

## Об использовании ИИ-инструментов при подготовке проекта

Промпты, код правил и узлов n8n прототипировались, тестировались и отлаживались с помощью
Claude Code: сначала на Python-прототипе против 576 реальных файлов корпуса (чтобы
убедиться, что правила действительно находят то, что заявлено в документах «Типичные
ошибки», а не шумят), затем перенесены в JavaScript для n8n. Это соответствует духу
самого курса — итоговое задание прямо предлагает «передать рутинное действие
искусственному интеллекту» и для Трека 1 отдельно просит промпты как часть сдачи.
